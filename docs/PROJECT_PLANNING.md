# k-calAI-RN Project Planning

## 제품 위치

`k-calAI-RN`은 독립적인 칼로리 계산 앱이 아니라, 헬스케어 애플리케이션 안에서 식단 기록을 돕는 이미지 기반 식단 분석 기능이다.

사용자에게 보여야 하는 핵심 가치는 "사진 한 장으로 식사 내용을 빠르게 기록하고, 예상 칼로리와 참고 정보를 확인한다"이다. 따라서 앱의 화면과 문구는 AI 모델 데모보다 건강 관리 루틴의 일부처럼 느껴져야 한다.

## 대상 사용자

- 식사 기록을 꾸준히 남기고 싶지만 직접 음식명과 칼로리를 입력하는 과정이 번거로운 사용자
- 다이어트, 체중 관리, 운동 기록과 함께 섭취량을 대략적으로 확인하려는 사용자
- 정확한 영양 분석보다 빠른 추정과 기록 보조를 우선하는 사용자

## 해결하려는 문제

- 사용자는 식사 기록 입력 과정이 길면 기록을 포기하기 쉽다.
- 음식명, 1인분 기준, 칼로리 수치를 직접 검색하는 흐름은 모바일에서 피로도가 높다.
- 헬스케어 앱의 식단 기록은 정밀한 의료 판단보다 반복 가능한 기록 경험이 중요하다.

이 프로젝트는 사진 입력, 음식 후보 확인, 예상 칼로리 설명 확인까지의 흐름을 짧게 만들어 식단 기록의 진입 장벽을 낮춘다.

## 현재 기능 범위

- 카메라 촬영 또는 앨범 선택으로 음식 사진 입력
- `multipart/form-data`의 `file` 필드로 이미지 서버 업로드
- `/api/predict` 응답의 `predictions` 배열을 신뢰도 순으로 표시
- 사용자가 음식 후보를 선택하면 `/api/gpt-predict`로 칼로리 설명 요청
- 서버 연결 주소를 `EXPO_PUBLIC_CALORIE_API_URL`, `EXPO_PUBLIC_CALORIE_DETAIL_API_URL`로 교체 가능

## 핵심 사용자 흐름

1. 사용자가 식사 화면에서 사진을 촬영하거나 앨범에서 선택한다.
2. 앱이 이미지를 분석 서버로 업로드한다.
3. 앱은 상위 음식 후보와 신뢰도를 보여준다.
4. 사용자가 가장 적절한 음식 후보를 선택한다.
5. 앱은 선택된 음식명 기준으로 예상 칼로리와 간단한 영양 참고 정보를 보여준다.
6. 향후 상위 헬스케어 앱에서는 이 결과를 식단 기록, 일일 섭취량, 운동/체중 데이터와 연결한다.

## 화면 기획

### 분석 탭

- 음식 사진 입력을 시작하는 주 화면
- 분석 전에는 촬영/앨범 선택과 서버 연결 상태를 보여준다.
- 분석 후에는 예측된 음식 후보 목록과 신뢰도를 보여준다.
- 후보 선택 후에는 예상 칼로리 설명을 같은 흐름 안에서 보여준다.

### 상태 탭

- 현재 분석 파이프라인의 연결 상태와 API endpoint를 확인하는 개발/진단용 화면
- 제품화 단계에서는 일반 사용자용 탭이 아니라 설정, 진단, 개발자 정보 영역으로 이동하는 것이 자연스럽다.

## API 의존성

앱은 `kcalAI-model` 서버를 기능 서버로 사용한다.

| 목적 | 메서드 | 경로 | 요청 | 응답 |
| --- | --- | --- | --- | --- |
| 음식 이미지 분류 | `POST` | `/api/predict` | `multipart/form-data`, `file` | `{ predictions: [{ label, score }] }` |
| 칼로리 설명 생성 | `POST` | `/api/gpt-predict` | `{ text, max_tokens }` | `{ response_text }` |

API 계약이 바뀌면 앱과 서버를 같은 작업 단위에서 함께 수정한다.

## MVP 기준

- 사용자가 사진을 선택하고 분석 요청을 보낼 수 있다.
- 서버 장애, 권한 거부, 응답 스키마 오류가 사용자에게 이해 가능한 메시지로 표시된다.
- 음식 후보는 최소 3개까지 표시할 수 있다.
- 칼로리 설명은 한국어로 짧고 명확하게 표시된다.
- 로컬 개발에서 iOS, Android 에뮬레이터, Web의 서버 주소 차이를 처리한다.

## 제품화 전 보완 사항

- 식단 기록 저장 기능 추가
- 사용자가 후보 음식명을 직접 수정하는 기능 추가
- 1인분 기준, 양 조절, 음식 조합 입력 지원
- 칼로리 결과의 출처와 추정치 고지 문구 추가
- 헬스케어 앱의 일일 섭취량, 목표 칼로리, 운동 기록과 연결
- 사용자 이미지 업로드와 저장 여부에 대한 개인정보 정책 정리

## 비목표

- 의료 진단 또는 질병 관리 판단을 제공하지 않는다.
- 음식 이미지 하나만으로 정확한 중량과 영양소를 확정하지 않는다.
- AI 모델 성능 비교 데모를 앱의 주요 사용자 경험으로 노출하지 않는다.

## 성공 지표

- 사진 선택부터 첫 분석 결과 확인까지의 시간
- 분석 실패율과 API 오류율
- 사용자가 예측 후보를 선택해 칼로리 설명까지 도달하는 비율
- 식단 기록 저장 기능이 추가된 이후의 기록 완료율
